How and where does this app store data?

This app stores the data in a Relational Database called SQLite. Ultimately, it's a single file that lives on your machine. You can find it in the app's folder at database/cover-database.db. Relational databases are far more powerful than a spreadsheet. It's why one-to-many relationships (e.g. one Cover to many Stamps) are hard to record in Excel, but easy to manage in this app.

How do I upgrade to a new app version correctly?

Every time you take a new version, it's a separately packaged application. The application expects a database to exist at database/cover-database.db. If it cannot find it, it makes that folder and file when the app loads. In other words, the new app version knows nothing about your old data. This is easy to solve: simply copy the database folder from your old app version into the new app version.

How do I back up my data?

Although the file type used by SQLite is very different from an Excel spreadsheet, that doesn't mean backups are hard or that your data will be tied to this app. You can make a copy of the cover-database.db file to store as a backup.

How do I view my data outside of the app?

To look at your data outside of the app, google "SQLite Viewer" and choose a program that suits you. Personally, I'm using VSCode with the SQLite Viewer extension by Florian Klampfer.

But I still want Excel Data...

If needed, I can add a way to convert the individual SQLite tables into comma-separated values (.csv) files. Excel CAN read .csv files directly. However, you'll want to resave .csv data as an Excel file. Otherwise, any formatting and equations you add to the .csv file in Excel will be lost when you close excel.
