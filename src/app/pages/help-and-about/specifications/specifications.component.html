App Header

  • Has a title: "Cover Database"
  • Has a nav bar
    • Links are highlighted when selected
    • "All Covers" leads to the view and search page
    • "Add" leads to the add cover page
    • "Help" leads to the help page

Page "All Covers"

  • Has a title: "All Covers"
  • Includes a "Search"" button
    • When clicked, does nothing
  • Includes an "Unsort"" button
    • When clicked, reverts covers to original sortation
    • When clicked, sets all header sorters to unsorted state
  • Includes a table of Covers
    • Column with Title "Id"
      • is sortable
      • Each row has the Cover Id
    • Column with Title "Name"
      • is sortable
      • Each row has the Cover Name
    • Column with Title "Year"
      • is sortable
      • Each row has the Cover Year
    • Column with Title "Origin"
      • is sortable
      • Each row has the Cover Origin
    • Column with Title "Destination"
      • is sortable
      • Each row has the Cover Destination
    • Column with Title "Details"
      • is not sortable
      • Each row has a view details button
        • When clicked, does nothing
  • Sort buttons
    • Have 3 states
      • Unsorted (grey up and down arrows)
      • Sort Ascending (blue up arrow)
      • Sort Descending (blue down arrow)
    • Begin in Unsorted state
    • First click moves to Ascending Sort State
    • Second click moves to Descending Sort State
    • Third click returns to Unsorted State
    • Sorts are applied from left to right
      • E.g. If Id and Year are sorted Ascending, the data is first sorted by Id, then resorted by Year. Year will be the main sort since it was used last.
  • On Load, gets all cover data from the database.
    • If load fails, it
      • returns no covers
      • Displays an error toast that covers could not be loaded
      • Logs the real error and query to the console

Page "Add"

  • Has a title: "Add New Cover"
  • Has a Cover Form
    • "Name" field is required text entry
    • "Year" field is non-required number entry
    • "Origin" field is a non-required text entry
    • "Destination" field is a non-required text entry
  • Has a save button
    • When clicked, it adds the form data to the database as a new Cover
    • Is disabled when form is invalid
    • Is disabled while processing a save
    • A progress spinner shows while processing
    • On success, a message toast displays
    • On Failure, an error toast displays
  • If the form has been touched and not submitted, and the user tries to navigate to another page the user is prompted
  • Cover Form is cleared and reset when loading the page
    • Prompt warns about lost progress
    • Prompt has "No" button
      • Prompt defaults focus to "No" button
      • "No" button stops navigation
      • "No" button closes prompt
    • Prompt has "yes" button
      • "Yes" button continues navigation
      • "Yes" button closes prompt
  • On Load, does nothing

Page "Help"

  • Has a title: "Help Topics"
  • Has an "Open Console"" button that opens dev tools
  • has sections that are revealed when clicked on
    • "Owner Manual" explains how data is stored, how to preserve copies, and upgrade considerations.
    • "Owner Manual" is open by default
    • "Specifications" explains what behavior to expect on every page
    • "Is it really "Broken"?" includes a rant about bugs vs. features
    • Selected section is highlighted in a different color
  • On Load, does nothing
  • Has no Submit

Toasts (Shared)

  • Message Toast
    • Has a blue background
    • Has an "Okay" button to dismiss itself on click
    • Auto dismisses after 3 seconds
  • Error Toast
    • Has a yellow background
    • Has an "Okay" button to dismiss itself on click
    • Never auto dismisses
  • Only one toast persists at a time
  • New toasts replace old toasts
